# EliteLineup.com — AI Scout Integration Guide

## What's included

```
backend/
  dk_client.py        # DraftKings API wrapper (player pools, injuries)
  ai_scout.py         # Claude-powered signal engine (hourly web scrape)
  slate_sync.py       # Background sync service (runs every hour)
  scout_routes.py     # New API endpoints + Express.js equivalents

frontend/
  components/
    ScoutPanel.tsx    # News ticker + boost cards (drops into both optimizers)
  hooks/
    useScoutBoosts.ts # Hook for managing scout boost state
  pages/
    ScoutDashboard.tsx  # New /scout page (standalone signal browser)
    INTEGRATION_DIFFS.ts  # Exact code changes for Optimizer + ProOptimizer
```

## Backend setup (5 steps)

### 1. Install dependencies
```bash
pip install requests anthropic
```

### 2. Set environment variables
```bash
ANTHROPIC_API_KEY=sk-ant-...   # required for Claude analysis
DK_API_KEY=                    # your DK API key (or leave blank for public endpoints)
```

### 3. Wire into your main server file
```python
# main.py or app.py
from slate_sync import SlateSyncService
from ai_scout import AIScout
from dk_client import DraftKingsClient

dk    = DraftKingsClient()
scout = AIScout()
sync  = SlateSyncService(db=get_db(), dk_client=dk, ai_scout=scout)
sync.start()   # starts background thread — runs every hour
```

### 4. Register the new API routes

**FastAPI:**
```python
from scout_routes import router as scout_router
app.include_router(scout_router)
```

**Express (TypeScript):**
Copy the Express block from the bottom of `scout_routes.py` into your routes file.

### 5. Update your existing player SELECT query

Add these columns to your `/api/slates/:id/players` response
(the sync service writes them; you just need to SELECT them):

```sql
boost_score          AS "boostScore",
boost_reason         AS "boostReason",
ownership_projection AS "ownershipProjection",
tags,
injury_status        AS "injuryStatus"
```

```

│  DraftKings API          Rotowire / ESPN / CBS Sports   │
│  (player pool, salaries)  (injury reports, lineup news) │

             │                            │
             ▼                            ▼
      dk_client.py                   ai_scout.py
      (every hour)                   (every hour)
             │                            │
             └──────────┬─────────────────┘
                        ▼
                 slate_sync.py
                 (background thread)
                        │
              ┌─────────▼──────────┐
              │  Database          │
              │  players.boost_score│

              └─────────┬──────────┘

```

## Key behaviors

- **Hourly refresh**: `slate_sync.py` runs every 3600 seconds in a background thread. Each refresh scrapes Rotowire + ESPN + CBS for the sport, sends the text to Claude, and upserts the signal-enriched player data into your DB.
- **Boost application**: In both optimizers, `scoutProjections` are merged into `playerProjections` before the LP solver runs. User-set custom projections always take priority over scout boosts.
- **"Apply Boosts" button**: The `ScoutPanel` has an optional button that pushes scout projections into `customProjections` state so they show in the UI and affect the displayed projected total.
- **ProOptimizer ownership**: `ownershipDeltas` from the scout are merged into the ownership model used by leverage mode and the fade system.
- **Auto-refresh timer**: The `ScoutStatusBar` and `ScoutPanel` both show a countdown. When it hits zero they invalidate the React Query cache and trigger a refetch of the signal data.
- **Force refresh**: The "Scan Now" button calls `POST /api/scout/refresh`, which fires `sync.force_sync(sport)` in a background task and returns immediately.

## Customizing boost weights

Edit `BOOST_WEIGHTS` in `ai_scout.py`:

```python
BOOST_WEIGHTS = {
  "starter_out":  +5.0,   # teammate starter ruled out
  "injury_opp":   +4.0,   # direct beneficiary
  "value_spike":  +1.5,   # salary value play
  "out":          -99.0,  # remove from pool
  # ... etc
}
```

## Troubleshooting

| Problem | Fix |
|---------|-----|
| No signals appearing | Check `ANTHROPIC_API_KEY` is set and valid |
| DK API 403 errors | The public DK endpoints don't need a key — check your `User-Agent` header |
| Boosts not showing in optimizer | Confirm your player SELECT includes `boost_score AS "boostScore"` |
| Scout not refreshing | Check the background thread is alive: `sync._thread.is_alive()` |
| Claude returning invalid JSON | Check logs — Claude sometimes wraps in markdown fences (ai_scout.py strips them) |
